# 爸妈一键通：代码到部署完成流程

## 1. 目标
这份文档只保留一条最短路径，适合下次复用。

目标是从本地代码到线上可访问链接，尽量压缩到 15 到 30 分钟内完成。

## 2. 本项目最终结果
- GitHub 仓库：[https://github.com/xeave/bama-yijiantong](https://github.com/xeave/bama-yijiantong)
- Vercel 线上地址：[https://bama-yijiantong.vercel.app/](https://bama-yijiantong.vercel.app/)

## 3. 最短路径
### 第一步：本地完成代码
- 在本地完成页面、样式、脚本和文档
- 先本地自测一遍：首页、表单、按钮、跳转、`localStorage`

### 第二步：初始化 Git
在项目目录执行：

```bash
cd /Users/zxx/Desktop/yin_fa
git init
git add .
git commit -m "init: create bama-yijiantong"
git branch -M main
```

### 第三步：创建 GitHub 仓库
- 打开 GitHub 新建仓库页
- 仓库名建议统一用短、好记、全小写加中划线
- 本项目使用：`bama-yijiantong`
- 建议先建 `Private`

### 第四步：连接远端并推送
优先使用 `SSH`，会比 `HTTPS` 少一次 token 或密码阻塞。

```bash
git remote add origin git@github.com:xeave/bama-yijiantong.git
git push -u origin main
```

如果机器上还没有 GitHub SSH，可先配置：
- 生成或复用 `~/.ssh/id_ed25519_github`
- 确认 `~/.ssh/config` 中 `github.com` 指向对应 key
- 测试：

```bash
ssh -T git@github.com
```

## 4. Vercel 最短部署流程
### 第一步：导入 GitHub 仓库
- 打开 Vercel 新建项目页
- 选择 `Import Git Repository`
- 导入 `bama-yijiantong`

### 第二步：如果提示安装 GitHub App
- 选择 `Only select repositories`
- 只勾选当前仓库 `bama-yijiantong`
- 完成 GitHub 密码确认

### 第三步：部署
- Project Name 保持 `bama-yijiantong`
- Application Preset 选 `Other`
- Root Directory 保持 `./`
- 直接点 `Deploy`

## 5. 为什么这次跑得比较久
这次主要慢在几个点：
- 中途需要从零创建 GitHub 仓库
- `HTTPS push` 卡在 GitHub 凭证，后面改成 `SSH`
- Vercel 需要额外安装 GitHub App
- GitHub 对安装应用触发了一次 `sudo mode` 密码确认
- 中间还穿插了名称统一、文案补充、页面优化和导航本地化

## 6. 下次怎么更快
如果以后再做同类项目，可以按下面方式压缩时间：

### 方案一：保留现有 GitHub SSH
- 继续使用已经可用的 `SSH`
- 避免再次走 `HTTPS` 凭证输入

### 方案二：保留 Vercel GitHub App 授权
- 以后只给新仓库勾选权限
- 不用再完整重装一次应用

### 方案三：固定命名规则
- 中文品牌名：适合传播
- 英文仓库名：短、全小写、中划线
- 例如：`bama-yijiantong`

### 方案四：固定交付模板
每次项目都保留这几份文档：
- `README.md`
- `docs/发布文案.md`
- `docs/上线发布清单.md`
- `docs/代码到部署完成流程.md`
- `docs/自定义域名方案.md`

## 7. 下次可直接复制的命令模板
```bash
cd /你的项目目录
git init
git add .
git commit -m "init: create your-project-name"
git branch -M main
git remote add origin git@github.com:你的用户名/your-project-name.git
git push -u origin main
```

- 手机端排版是否正常
