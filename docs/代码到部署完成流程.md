# 爸妈一键通：代码到部署完成流程

## 1. 目标
这份文档只保留一条最短路径，适合下次复用。

目标是从本地代码到线上可访问链接，尽量压缩到 15 到 30 分钟内完成。

## 2. 本项目最终结果
- GitHub 仓库：[https://github.com/xeave/bama-yijiantong](https://github.com/xeave/bama-yijiantong)
- GitHub Pages 目标地址：[https://xeave.github.io/bama-yijiantong/](https://xeave.github.io/bama-yijiantong/)

## 3. 最短路径
### 第一步：本地完成代码
- 在本地完成页面、样式、脚本和文档
- 先本地自测一遍：首页、表单、按钮、跳转、`localStorage`

### 第二步：初始化 Git
在项目目录执行：

```bash
cd /Users/zxx/Desktop/yin_fa
git init
git add .
git commit -m "init: create bama-yijiantong"
git branch -M main
```

### 第三步：创建 GitHub 仓库
- 打开 GitHub 新建仓库页
- 仓库名建议统一用短、好记、全小写加中划线
- 本项目使用：`bama-yijiantong`
- 如果你使用的是 `GitHub Free`，建议直接建 `Public`

### 第四步：连接远端并推送
优先使用 `SSH`，会比 `HTTPS` 少一次 token 或密码阻塞。

```bash
git remote add origin git@github.com:xeave/bama-yijiantong.git
git push -u origin main
```

如果机器上还没有 GitHub SSH，可先配置：
- 生成或复用 `~/.ssh/id_ed25519_github`
- 确认 `~/.ssh/config` 中 `github.com` 指向对应 key
- 测试：

```bash
ssh -T git@github.com
```

## 4. GitHub Pages 最短部署流程
### 第一步：确认仓库可发布
- 当前仓库已经准备好 `.github/workflows/pages.yml`
- 如果你使用的是 `GitHub Free`，确认仓库为 `Public`

### 第二步：推送代码
- 把当前代码推送到 `main`
- GitHub Actions 会自动触发 `Deploy GitHub Pages`

### 第三步：开启 Pages
- 打开 GitHub 仓库 `bama-yijiantong`
- 进入 `Settings`
- 打开 `Pages`
- 在 `Build and deployment` 中确认 `Source` 为 `GitHub Actions`

### 第四步：等待正式地址
- 打开仓库 `Actions`
- 等待 `Deploy GitHub Pages` 成功
- 成功后访问：`https://xeave.github.io/bama-yijiantong/`

## 5. 为什么这条路更省事
这次切到 GitHub Pages 之后，流程会更轻：
- 不需要再登录额外的部署平台
- 不需要安装 GitHub App
- 不需要单独维护 Vercel 配置
- 纯静态 HTML/CSS/JS 项目可以直接发布

## 6. 下次怎么更快
如果以后再做同类项目，可以按下面方式压缩时间：

### 方案一：保留现有 GitHub SSH
- 继续使用已经可用的 `SSH`
- 避免再次走 `HTTPS` 凭证输入

### 方案二：保留 GitHub Pages 工作流
- 继续复用 `.github/workflows/pages.yml`
- 新项目只需要改仓库名和内容，不用重搭部署

### 方案三：固定命名规则
- 中文品牌名：适合传播
- 英文仓库名：短、全小写、中划线
- 例如：`bama-yijiantong`

### 方案四：固定交付模板
每次项目都保留这几份文档：
- `README.md`
- `docs/发布文案.md`
- `docs/上线发布清单.md`
- `docs/代码到部署完成流程.md`
- `docs/自定义域名方案.md`

## 7. 下次可直接复制的命令模板
```bash

git add .
git commit -m "init: create your-project-name"
git branch -M main
git remote add origin git@github.com:你的用户名/your-project-name.git

```

- 手机端排版是否正常
