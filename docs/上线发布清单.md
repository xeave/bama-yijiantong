# 爸妈一键通上线发布清单

## 1. 当前项目已准备好的内容
- 首页入口：`index.html`
- 用药提醒页：`meds/index.html`
- 血压血糖页：`health/index.html`
- 大字计算器页：`calculator/index.html`
- 杭州常用网址导航页：`nav/index.html`
- 公共样式与脚本：`assets/`
- GitHub Pages 自动发布：`.github/workflows/pages.yml`
- 发布文案：`docs/发布文案.md`

## 2. 上传 GitHub Pages 前先确认
- 首页联系方式已是正式信息
- 微信复制按钮可正常使用
- 用药提醒页可以新增、删除、本地保存
- 血压血糖页可以新增、删除、本地保存
- 计算器可以正常计算
- 杭州导航页链接能正常打开
- 页面在手机宽度下排版没有明显错位
- 如果你使用的是 `GitHub Free`，仓库通常需要是 `Public`

## 3. GitHub Pages 发布步骤
推荐直接使用仓库里已经准备好的工作流。

1. 先把项目上传到 GitHub 并推送到 `main`
2. 打开仓库 `Settings`
3. 进入 `Pages`
4. 在 `Build and deployment` 中确认 `Source` 为 `GitHub Actions`
5. 打开仓库 `Actions`
6. 等待 `Deploy GitHub Pages` 工作流执行成功
7. 访问正式链接：`https://xeave.github.io/bama-yijiantong/`

## 4. GitHub Pages 配置说明

说明：
- 当前项目是纯静态站点，不需要 Node 构建流程
- `.github/workflows/pages.yml` 会直接发布当前仓库内容
- 4 个工具页已改成目录式路由，GitHub Pages 下可以直接使用 `/meds/`、`/health/`、`/calculator/`、`/nav/`

## 5. 部署完成后重点检查
- 首页是否能正常打开
- 首页“点击复制”按钮是否能复制微信号
- `/meds/` 页面是否能正常保存提醒
- `/health/` 页面是否能正常保存记录
- `/calculator/` 页面是否能正常计算
- `/nav/` 页面是否能正常打开杭州官方入口

## 6. 部署后建议访问的路径
- 首页：`https://xeave.github.io/bama-yijiantong/`
- 用药提醒：`https://xeave.github.io/bama-yijiantong/meds/`
- 血压血糖：`https://xeave.github.io/bama-yijiantong/health/`

- 杭州常用网址导航：`https://xeave.github.io/bama-yijiantong/nav/`

## 7. 手机端自测清单
- 标题是否足够清晰
- 按钮是否足够大
- 表单是否容易输入
- 页面是否需要横向滚动
- 联系方式是否清楚可见
- 杭州导航链接是否容易点击

## 10. 这版项目的边界
- 数据只保存在浏览器本地
- 换设备或清缓存后数据不会保留
- 医院、医保、社保入口后续可能会调整
- 后续如果扩展到其他城市，需要替换杭州本地导航
